Este nombre se utiliza como clave cuando se envían los datos del formulario al servidor. 2. value Descripción: Establece el valor inicial del control de formulario. Para los botones, define el texto que aparece en el botón. 3. placeholder Descripción: Proporciona una pista o indicación al usuario sobre lo que debe ingresar en el campo de texto. 4. required Descripción: Indica que el campo de formulario es obligatorio y debe ser completado antes de enviar el formulario. 5. readonly Descripción: Hace que el campo de formulario sea de solo lectura. El usuario no puede modificar su valor. 6. disabled Descripción: Desactiva el control de formulario, lo que impide que el usuario interactúe con él. Los datos de los controles desactivados no se envían al servidor. 7. maxlength Descripción: Establece el número máximo de caracteres que el usuario puede introducir en un campo de texto. 8. pattern Descripción: Define una expresión regular que el valor del campo de texto debe coincidir para que el formulario se considere válido.
